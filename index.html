💡 역할

애플은 저의 첫 프로젝트로 팀원으로서 다양한 역할에 기여할 수 있었습니다.
팀원 모두가 풀스택 기술을 활용할 수 있었기 때문에, 함께 주제를 정하고 테이블 구조를 구상한 뒤 각자의 역할을 나누어 프론트와 백엔드 모두를 담당하기로 결정하였습니다.

많은 동물 병원 사이트와 입양 플랫폼이 존재하지만 급한 상황에 사람들이 접근하기 힘들다는 점,
사용자에게 조금 무거운 느낌을 주다보니 빠른 공유와 정보전달이 SNS보다는 쉽지 않다는 점 등의 문제점들이 있다고 생각했습니다.
또한, 아직도 많은 사람들이 반려동물의 임시보호글, 도움을 청하는 글, 산책 등의 소모임 글들을 다른 목적의 커뮤니티 사이트(ex. 당X마켓, X브리타임)에 올리는 것을 보았습니다.
이러한 점들을 생각하며 ‘반려인들을 위한 통합 사이트' 라는 주제를 적극적으로 제안하였고 팀원들의 다양한 의견을 모아 주제로 선정하게 되었습니다.

주제 논의 단계부터 소모임 기능에 대한 아이디어를 지속적으로 표출해왔기에, 저는 반려인들을 위한 모임 게시판 기능을 담당하게 되었습니다.
스케줄 위젯, 월별 모임 등의 아이디어를 계속해서 떠올리며 사용자의 편의를 고려한 방향으로 구현하였습니다.

제 파트 구현이 일찍 끝난 뒤엔 레퍼런스를 참고하여 전체적인 메인 디자인 부분을 손보았으며 푸터를 제작하였습니다.
발표 준비를 위해 PPT를 미리 제작하여 프로젝트 기간을 조금 더 단축할 수 있었습니다.

또한, 동물이라는 주제에 어울리도록 전체적으로 동글동글한 폰트와 디자인 요소를 적용하였습니다.

📌 모임 게시판 페이지

블로그 느낌을 주기 위해 위젯 디자인으로 설계하였습니다.
날짜가 중요한 사이트이니만큼 달력, 월, D-day 등으로 계속해서 사용자에게 날짜를 보여주는 것이 목표였습니다.
참여하기, 참여취소 버튼의 레이아웃은 휴대용 어플 같은 느낌을 주기 위해 부드럽게 디자인 하였습니다.

📌 모임 작성 페이지

화려한 다른 페이지와 차별화를 위해 색상 사용을 줄여 사용자에게 차분하고 집중하기 쉬운 경험을 제공하였습니다.

🛠️ 모임 게시판 페이지 기능

해당 달의 모임과 참여인원을 확인할 수 있습니다.

사용자는 총 3개의 모임에 참가신청을 할 수 있으며 날짜는 중복될 수 없습니다.
사용자가 참여하기 및 참여취소를 누르면 비동기 방식 요청을 통해 이달의 모임의 인원수, 게시글의 멤버칸, 내 스케줄 위젯 총 3가지의 클라이언트 측 변동이 있으며 사용자, 게시글 총 2가지의 서버 측 변동이 있습니다.

제목/내용/작성자/제목 또는 내용/모임날짜로 필터링하여 검색 가능합니다.

글 작성 버튼을 누르면 글쓰기 페이지로 이동합니다.

🛠️ 상세 게시판 페이지 기능

앞선 모임 게시판에서 해당 게시글을 열람하면 페이지 이동 및 조회수가 증가합니다.
동일 구조에 댓글 작성, 수정, 삭제기능이 추가되었습니다.
페이지를 분리한 이유는 댓글들을 다른 페이지에서 보여주어 모임 페이지에서 모임에 집중하게 하기 위함입니다.

🛠️ 모임 작성 페이지 기능

모임을 직접 설정, 수정, 삭제할 수 있습니다.
위치, 인원수는 필수 입력값으로 유효성 검사 포함하였습니다.
글 작성자는 모임 작성 시 자동으로 해당 모임에 참가됩니다.

이 프로젝트에서 팀장 역할을 맡아 전반적인 방향성을 설정하고 팀원들과 협업하며 조금 더 폭넓은 참여로 프로젝트를 이끌었습니다.

초기 기획부터 시작하여 풀스택 개발자로 참여하였으며 다양한 기능과 API를 활용하는 데 중점을 두었습니다.
초기 주제는 스포츠 경기 정보를 외부 API를 통해 제공하는 것이었으나, 금전적인 제약으로 경기 데이터를 직접 생성하는 방식으로 방향을 수정하였습니다.
이로 인해 배포 계획을 수정하고 데이터베이스에 직접 입력해야 하는 데이터를 최소화, 기능 구현에 집중했습니다.

구단 정보 및 굿즈 게시판의 데이터들은 최소한으로 생성하여 기능 구현 정보들로 활용하였습니다.
이벤트 게시판과 관리자 페이지에서의 이벤트 생성 및 당첨자 뽑기 부분을 담당하였으며
경품 SHOP, 당첨자 SHOP, 결제 / 환불 / 배송 페이지를 구현하였습니다.
마이페이지에서는 사용자의 구매내역, 배송내역 부분과 위시리스트, 장바구니, 적립금/포인트 페이지를 담당하여 구현하였습니다.

감추기, 삭제, 게시판 이동을 선택하여 수행할 수 있습니다.

🛠️ 관리자 페이지, 이벤트 게시판 페이지 - 이벤트 기능

생성된 경기에 이벤트를 추가할 수 있습니다.
이벤트는 룰렛, 단어 맞추기 총 두가지로 경품 SHOP에 올라와 있는 경품과 적립금 금액을 선택하여 이벤트를 생성합니다.
관리자는 이벤트 게시판으로 이동, 글을 작성하여 사용자에게 공지하며 경기가 종료된 후 종료된 경기의 당첨자를 추첨 할 수 있습니다.
이벤트별로 당첨조건이 다르며 유저 추첨이 완료되면 이벤트 게시판에 공지합니다. 게시판은 [이벤트], [공지사항] 으로 나누어 작성하며 메인 페이지에서 필터링하여 확인할 수 있습니다. 공지 시 당첨자의 아이디는 마스킹 처리되어 보여집니다. 게시판은 조회수순, 최신순, 오래된순 별로 필터링 가능합니다.
이벤트 종류가 룰렛인 경우 이긴 팀의 댓글만 가져오며 중복 가능하여 댓글을 많이 달 수록 참여자에게 유리합니다. 댓글 방식일 경우엔 승패에 상관없이 모든 댓글을 가져오며 맞춰진 댓글만 필터링하고 중복값을 제거하여 추첨합니다.

🛠️ 경품 SHOP, 당첨자 SHOP 기능

경품 SHOP에서 사용자는 각 상품의 옵션을 선택한 후 바로결제 / 위시리스트에 담기 / 장바구니에 담기 중 하나를 선택할 수 있습니다.
당첨자 SHOP은 이벤트 경품별로 선택하여 수령할 수 있습니다. 적립금을 수령하면 10일동안 유효하며, 상품을 수령하면 배송 페이지로 이동합니다.
user 테이블의 칼럼에 역슬래시로 묶여있는 이벤트 번호들을 바탕으로 서버는 경품 정보를 확인하여 클라이언트측으로 보여주며 경품을 수령 시 각 번호의 앞에 -를 붙여 미수령 경품과 구분해줍니다.

🛠️ 결제 / 환불 / 배송 페이지 기능

사용자는 결제정보를 확인하여 포인트 결제, 간편 결제, 적립금 결제 중 하나의 수단으로 결제할 수 있습니다. 포인트 결제는 충전방식, 간편 결제는 토스페이먼츠의 결제위젯 테스트 API, 적립금 결제는 보유 적립금 차감으로 이루어집니다. 결제 전 적립금 사용을 선택할 수 있으며 최소 3000원 이상부터 사용가능합니다. 성공적으로 결제를 마치면 마이페이지로 이동하며 결제금액의 10%만큼 결제 적립금이 충전됩니다.
환불을 원하는 경우 사용자는 환불수단을 선택하여야하며 단순 변심일 경우만 제외하고 전액 환불됩니다.
배송 페이지에서는 단순 배송만 진행됩니다. 사용자는 본인의 주소를 다음 주소찾기 API를 통해 직접 설정할 수 있으며 상세주소와 배송메모도 선택 가능합니다.

🛠️ 마이페이지 기능

주문/배송조회 페이지에서 사용자의 주문목록을 확인할 수 있습니다. 주문 상태 변화는 날짜가 지남에 따라 자동으로 변경됩니다.
위시리스트와 장바구니에서는 담긴 상품 목록을 확인합니다. 위시리스트에서는 바로결제 / 장바구니에 담기 / 위시리스트에서 삭제 중 하나를 선택할 수 있습니다.
장바구니에서는 해당 상품의 옵션을 변경, 삭제할 수 있으며 결제할 상품을 선택하여 결제 페이지로 넘어갈 수 있습니다. 사용자는 보유 적립금과 포인트를 카테고리별로 확인할 수 있습니다. 최초 가입 시 가입 축하 적립금 5000P를 받으며 이벤트 적립금은 시간이 지나면 소멸합니다.
